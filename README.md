# تقويمي — رزنامة كويتية ذكية

منصة رزنامة كويتية Mobile-first تتيح للموظفين والمواطنين إنشاء رزنامة مخصّصة، إضافة المناسبات والنوتات، وطباعتها على A4/A3 أو استخدامها على الهاتف. العطل الرسمية الكويتية تظهر تلقائيًا، وقابلة للتعديل من المستخدم.

مشروع **مستقل** في مستودعه الخاص `taqwimi`، يُنشر تلقائيًا على GitHub Pages تحت المسار `‎/taqwimi/‎`.

## الستاك

- **Next.js 15 (App Router) + TypeScript + Tailwind CSS**
- **date-fns** لأدوات التواريخ
- بدون قاعدة بيانات — كل شيء في `localStorage`
- RTL عربي كامل (`dir="rtl"`, `lang="ar"`)، خط Cairo
- وضع ليلي/نهاري (يُحفظ محليًا ويحترم تفضيل النظام)، وخلفيات كويتية مائية اختيارية لكل شهر

## قواعد التواريخ (صارمة)

- المنطقة الزمنية دائمًا **Asia/Kuwait** (UTC+3 ثابت، بلا توقيت صيفي).
- الأيام تُخزَّن كنصوص `YYYY-MM-DD` وتُقارن نصيًا — لا `new Date('YYYY-MM-DD')` ولا `toISOString()` للعرض.
- «اليوم» عبر `Intl.DateTimeFormat('en-CA', { timeZone: 'Asia/Kuwait' })` على العميل فقط.
- اليوم من الأسبوع بخوارزمية **Sakamoto** (حساب صحيح صرف، مستقل عن المنطقة الزمنية).
- نهاية الأسبوع: الجمعة والسبت. بداية الأسبوع الافتراضية: الأحد (قابلة للتغيير).
- شبكة الشهر ثابتة **6×7**، وفي RTL أول يوم يبدأ من اليمين.

## تتبع ١٨٠ يوم (ميزة اختيارية)

تتبع استحقاق **الأعمال الممتازة** لموظفي الجهات الحكومية بالكويت. تُفعَّل من الإعدادات (الافتراضي مطفأة — وعند الإطفاء لا يظهر أي أثر لها في التطبيق).

- **النموذج خصمي**: كل يوم عمل منقضٍ (أحد–خميس، غير عطلة) يُحسب دوامًا تلقائيًا، والمستخدم يسجّل الإجازات فقط (إجازة/مرضية/طارئة/أخرى) من ورقة اليوم.
- **الحساب**: المنجز = أيام العمل المنقضية − الإجازات؛ المتاح = أيام العمل من الغد حتى ٣١ ديسمبر − الإجازات المجدولة؛ رصيد الأمان = المتاح − المتبقي للهدف (آمن ≥ ٢٠، انتبه ≥ ١٠، خطر أقل).
- **أيام العمل تُشتق من محرك عطل تقويمي نفسه** (`effectiveHolidaysForYear`) — أي تعديل/حذف/إضافة من «إدارة العطل» ينعكس على العدّاد فورًا.
- بطاقة ملخّص أعلى مفتاح الرموز (دائرة تقدّم + المتاح/المتبقي/رصيد الأمان)، ونقطة حمراء على الأيام المسجّلة إجازة (شاشة فقط — **الطباعة لا تتأثر إطلاقًا**).
- البيانات في `localStorage` (`track180Days`) وتدخل في «تصفير البيانات». المحرك في `lib/track180.ts` واختباراته في `lib/__tests__/track180.test.ts`.

## التطوير

```bash
npm install
npm run dev      # http://localhost:3000/taqwimi  (أو NEXT_PUBLIC_BASE_PATH="" للجذر)
npm run test     # اختبارات وحدة للمحرّك والعطل
npm run build    # تصدير ثابت إلى out/
```

## النشر — GitHub Pages (مستودع مستقل)

«تقويمي» يُنشر من مستودعه الخاص على GitHub Pages تحت المسار `‎/taqwimi/‎`.

الآلية تلقائية عبر `.github/workflows/deploy-pages.yml`: عند كل دفع إلى `main` يُبنى التطبيق (تصدير ثابت إلى `out/`) ويُنشر إلى فرع `gh-pages`.

خطوة واحدة لمرة واحدة بعد أول دفع: من **Settings → Pages** في المستودع، اختر **Deploy from a branch** ثم الفرع **`gh-pages`** والمجلد `/ (root)`.

الرابط بعدها: `https://bowaleedpc90-cell.github.io/taqwimi/`

التطبيق كامل على العميل (`localStorage`) وبدون خادم. لتشغيله محليًا على الجذر بدون مسار أساسي:

```bash
NEXT_PUBLIC_BASE_PATH="" npm run dev
```

## الخلفيات الشهرية (مُزالة)

كانت هناك ميزة «زخرفة الشهر» — خلفية كويتية مائية خفيفة (watermark) خلف الرزنامة على الشاشة والطباعة، مع زر تشغيل/إطفاء في إعدادات الطباعة، وصور في `public/monthly-art/`. **أُزيلت الميزة بالكامل** (الشاشة والطباعة صارتا نظيفتين بلا أي خلفية صور) بناءً على طلب المالكة.

**لاسترجاعها من git لاحقًا لو احتجنا:** الميزة كانت موجودة حتى الكوميت `c2cf6d3` (نهاية المرحلة ٣). لاستعادتها:

```bash
# استعادة الصور والمكوّن من كوميت ما قبل الإزالة
git checkout c2cf6d3 -- public/monthly-art components/MonthWatermark.tsx
# ثم راجع فرق app/globals.css و components/{CalendarGrid,PrintMonth,PrintPreview,DayCell}.tsx
git show c2cf6d3 -- app/globals.css components/CalendarGrid.tsx components/PrintMonth.tsx components/PrintPreview.tsx
```

(المكوّن `MonthWatermark` كان يعرض الصورة بـ `object-cover` وشفافية منخفضة عبر صنف `.month-wm` في `globals.css`.)
